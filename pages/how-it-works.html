How It Works

The Technology Behind SecretNote

SecretNote is built on three principles: strong encryption, zero server knowledge, and guaranteed destruction.

Step by Step: What Happens When You Create a Note

1

You Write Your Note

Your note stays in your browser. Nothing is sent to any server at this point.

2

Browser Generates an Encryption Key

Your browser creates a random 256-bit AES key using the Web Crypto API — a cryptographically secure random number generator built into every modern browser.

3

Note Is Encrypted Locally

Your browser encrypts the note text using AES-256-GCM with a random 96-bit IV. The result is an unreadable blob of encrypted bytes.

4

Encrypted Note Gets an ID and Is Stored

The encrypted blob is saved to browser localStorage under a random 12-byte ID. The key used to encrypt it is NOT stored anywhere — only in your URL.

5

A Link Is Generated With the Key in the Fragment

Your link looks like: secretnote.com/read.html?id=abc123#decryptionkeyhere. The part after # is the decryption key. Browser URL fragments are never sent to servers — only your recipient's browser sees it.

6

Recipient Opens the Link

Their browser reads the encrypted note from storage using the ID, then uses the key from the URL fragment to decrypt it locally. The decryption happens entirely in their browser.

7

Note Is Destroyed Immediately

The moment the note is decrypted and displayed, it is deleted from storage. No copy remains anywhere. Anyone clicking the link again sees "Note Not Found."

8

Time Expiry (if set)

If you set a time limit (1 minute, 5 minutes, etc.), the expiry timestamp is stored with the note. Any attempt to open an expired note results in immediate deletion and an expiry message.

Why Is the Key in the URL Fragment?

The URL fragment (the #key part) is a privacy feature of the web: it is processed entirely by your browser and is never included in HTTP requests to servers. When you share a link like secretnote.com/read.html?id=abc#KEY, our server only sees secretnote.com/read.html?id=abc — never the key.

This means even if someone had full access to our server logs, they could not decrypt any notes. The key only ever exists in your URL and the recipient's browser memory during decryption.

Important Limitation

Be careful how you share the link. If you paste it into a chat app that generates link previews, the preview request might reveal the full URL (including the fragment in some implementations). For maximum security, share the link directly rather than through preview-generating platforms.
