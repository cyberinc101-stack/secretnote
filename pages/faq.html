Frequently Asked Questions

Everything you need to know about sending self-destructing encrypted notes with SecretNote.

General Questions

What is a self-destructing note?

A self-destructing note is a message that automatically deletes itself after being read — or after a set time limit. Unlike email or text messages which remain in inboxes indefinitely, a SecretNote exists only until the moment your recipient reads it, then it is gone forever.

Is SecretNote completely free?

Yes. SecretNote is completely free with no account required, no storage limits per note, and no paid tiers. The site is supported by advertising.

Do I need to create an account?

No. SecretNote requires no account, no email, and no personal information of any kind. Just write, create, and share.

Who uses self-destructing notes?

SecretNote is used by professionals sharing passwords and credentials, developers sharing API keys and tokens, businesses sending confidential information, journalists protecting sources, individuals sharing private personal messages, and anyone who needs to share sensitive information without leaving a permanent digital trail.

Privacy & Security

Does SecretNote store my notes on a server?

Encrypted note data is stored in your browser's localStorage — not on any external server. The encrypted ciphertext has no value without the decryption key, which only exists in the shareable URL fragment. Even if someone gained access to the storage, they cannot read the note without the key.

Can SecretNote read my notes?

No. Your note is encrypted in your browser before anything is stored. The decryption key exists only in the URL you share. SecretNote never has access to the key and therefore can never decrypt your notes.

What encryption does SecretNote use?

SecretNote uses AES-256-GCM encryption — the same standard used by banks, governments, and military organisations worldwide. Keys are generated using the Web Crypto API's cryptographically secure random number generator.

Can someone intercept the note in transit?

The note itself is never transmitted — only the encrypted ciphertext is stored locally. The decryption key is in the URL fragment, which most browsers do not include in server requests. However, some chat platforms or link-preview services may process the full URL. For maximum security, share links through channels that do not generate link previews.

Can someone take a screenshot of my note?

SecretNote cannot technically prevent screenshots or photos of the screen. Our protection is against digital storage and forwarding — the note cannot be forwarded as a link because it no longer exists after reading. For truly sensitive information, keep notes brief and use the password protection feature.

Expiry & Timing

What expiry options are available?

You can set a note to expire: after reading only (default), after 1 minute, 5 minutes, 10 minutes, 20 minutes, 30 minutes, 1 hour, 24 hours, 7 days — or enter any custom duration. Custom durations can be set in minutes, hours, or days up to 365 days.

What happens if the link expires before the recipient reads it?

The note is automatically deleted and the recipient sees an "expired" message. They cannot tell whether the note was read by someone else or simply expired. For time-sensitive credentials, choose a short expiry window and let your recipient know to open it promptly.

Can I extend the expiry time after creating a note?

No. Once a note is created, its expiry cannot be changed. If you need a longer expiry, create a new note with the desired time limit.

What is "Destroy immediately on open"?

By default, recipients see a confirmation screen before the note is revealed — giving them a chance to prepare. If you check "Skip confirmation", the note is decrypted and destroyed the moment the link is opened, with no confirmation step. Use this when you want instant destruction.

Passwords & Options

Should I password-protect my notes?

For highly sensitive information, yes. Even though notes are already encrypted, adding a password provides a second layer of protection — if the link is accidentally shared with the wrong person, they cannot read the note without the password. Share the password through a completely separate channel (e.g., by phone if the link was sent by email).

What if I forget to read my own note link?

The note is stored in the browser where you created it until it is opened or expires. If you need to destroy it before the recipient reads it, use the "Destroy Now" button on the link creation screen. Once you navigate away and lose the link, there is no way to recover or destroy the note manually — it will expire on schedule.
