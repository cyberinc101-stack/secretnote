How to Send a Password Securely Online

Email and chat apps store passwords permanently on servers. Here is the safer alternative — and why it matters.

At some point everyone needs to send a password to someone else — a colleague, a client, a family member. Most people send it over email, Slack, or WhatsApp without thinking twice. The problem is that those platforms archive everything. That password sits in someone's inbox, on a corporate server, and in backups, indefinitely.

Why sending passwords over email is dangerous

Email is the most common way passwords get exposed after a breach. When an email account is compromised — whether by phishing, credential stuffing, or a data breach at the provider — attackers search the inbox for passwords. They almost always find them.

The safest way to send a password online

A self-destructing encrypted note is the most practical secure method for sharing a password. The note is encrypted in your browser, stored temporarily as unreadable ciphertext, and permanently deleted the moment the recipient opens it. There is nothing left to breach.

How SecretNote works for password sharing

  1. Paste the password into SecretNote. It is encrypted in your browser using AES-256-GCM before anything leaves your device.
  2. Click Create Note. You receive a one-time link containing the decryption key after the # symbol — a part of the URL that is never sent to any server.
  3. Send the link over email, chat, or any channel. If it is intercepted at this point, an attacker sees only an unread note — not the password itself.
  4. The recipient opens the link, the password is decrypted locally in their browser, and the note is permanently deleted from storage. The link becomes dead immediately.

Extra protection: add a password to the note

For highly sensitive credentials, use SecretNote's optional passphrase feature. The note itself is encrypted, and a second passphrase is required to read it. Send the one-time link over email and the passphrase over a different channel — a phone call, text, or in person. An attacker intercepting either channel alone gets nothing.

Set a time limit for extra safety

If the recipient does not open the note within a set window, it self-destructs automatically. Useful options include 1 hour (for immediate handoffs), 24 hours (for end-of-day access), or 7 days (for contractors or delayed onboarding). After expiry, the link is dead and the ciphertext is deleted — even if no one ever opened it.

Frequently asked questions

Can SecretNote see my password?

No. Encryption happens in your browser before anything is transmitted. The decryption key exists only in the URL fragment, which browsers never send to servers. SecretNote's server receives and stores only encrypted ciphertext it cannot read.

What if the link is intercepted before the recipient opens it?

The attacker would need to open the link before your recipient does. If that happens, your recipient gets an "already read" error — a clear signal that something went wrong. For critical credentials, use the passphrase feature as an additional layer.

Is there a limit on what I can send?

Notes can contain up to 50,000 characters — more than enough for any password, API key, certificate, or block of credentials.

Do I need an account?

No account, no signup, no email address required. Create a note and share the link. That is it.
