The key difference: where the encryption key lives

This is the most important technical distinction. In SecretNote, the AES-256-GCM decryption key is placed in the URL fragment — the part after the # symbol. Browsers never include URL fragments in HTTP requests, which means our server never sees the key. We physically cannot decrypt your note.

Privnote's encryption operates differently: the server participates in the decryption process, which means the operator has the technical ability to read notes. For most casual use this may not matter, but for anything sensitive — passwords, API keys, private messages — true zero-knowledge encryption is the right choice.

When Privnote is still fine

For low-sensitivity content — surprise party plans, trivia answers, temporary notes — Privnote works well and has a long track record. The encryption distinction matters most when you are sharing credentials, keys, financial information, or anything you would not want a server operator to be able to read.
