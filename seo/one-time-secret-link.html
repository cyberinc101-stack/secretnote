A one-time secret link is a URL that contains — or points to — a piece of sensitive information. The link works exactly once. After it is opened, the content is permanently destroyed and the link becomes dead. Anyone who tries to open it a second time sees nothing.

SecretNote generates one-time secret links for free, with AES-256 encryption baked in, directly from your browser.

How a one-time link is structured

SecretNote one-time links are made of two parts:

https://secretnote-six.vercel.app/read.html?id=a7f3c9d2e1b4#k=9f2a8c1d7e3b...
Domain + note ID (sent to server — identifies the encrypted blob) Decryption key in URL fragment (never sent to server — invisible to us)

The note ID lets the server retrieve the correct encrypted ciphertext. The fragment key (after #) decrypts it — but browsers never include URL fragments in HTTP requests, so the server never sees the key. This is what makes it truly zero-knowledge.

One-time links vs regular links

A regular link points to content that stays there indefinitely. Anyone with the link can open it now, in a year, or a decade from now. A one-time link is fundamentally different — it is a single-use credential that expires on first access.

This matters because links get copied, forwarded, indexed by browsers, and stored in email threads. A one-time link has zero value after its first use. Even if the URL leaks, an attacker who finds it in someone's email sees a dead link.

Add time-based expiry on top

One-time links already expire on first use. You can add a second expiry condition — time. Set a window from 1 minute to 7 days. If the recipient does not open the link within that time, it self-destructs automatically, even if it was never read. This covers scenarios where a link is sent to the wrong person or the recipient is delayed.

Frequently asked questions

What happens if I open my own link before the recipient does?

The note is destroyed and the link goes dead. The recipient will see a "note not found" message. Always copy the link and send it without opening it yourself — or use the Delete Note Now button if something goes wrong.

Can I generate multiple one-time links from the same content?

Yes — create a separate note for each recipient. Each creation generates an independent link pointing to its own encrypted copy.

Is the one-time link secure to send over email?

Sending the link over email is fine — the link itself reveals nothing without the decryption key in the fragment. The real risk is someone opening the link before your intended recipient does. For critical secrets, use the passphrase feature as a second factor.

How long is the link valid?

By default, a link is valid until first use (no time limit). You can optionally set a maximum time window from 1 minute to 7 days.
